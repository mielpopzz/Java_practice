Практика #1:
try {
  ...
}
НЕТ, такая конструкция НЕ допустима в Java.
try должен сопровождаться как минимум одним catch блоком или finally блоком.

Практика #2:
try {
  ...
} finally {
  ...
}
ДА, такая конструкция допустима.
Это полезно для гарантированного освобождения ресурсов, даже если исключение не перехватывается.

Практика #3:
try {
  ...
} finally {
  ...
} finally {
  ...
}
НЕТ, такая конструкция НЕ допустима.
Может быть только один finally блок для каждого try.
