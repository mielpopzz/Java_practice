Практика #1
Интерфейс, объявленный как interface Interface { ... } имеет package-private (default) область видимости. Это означает:

Доступен только внутри того же пакета

Не доступен из других пакетов

Не требует явного указания модификатора (default)

Практика #2
Интерфейс, объявленный как public interface Interface { ... } имеет public область видимости. Это означает:

Доступен из любого места (любого пакета)

Это наиболее распространенный способ объявления интерфейсов

Ограничений на использование нет

Практика #3
Интерфейс, объявленный как protected interface Interface { ... } - это НЕКОРРЕКТНОЕ объявление и вызовет ошибку компиляции. Причины:

Интерфейсы не могут быть protected

Интерфейсы могут быть только: public или package-private (default)

Защищенный доступ (protected) имеет смысл только для членов класса, но не для самого интерфейса

Практика #4
Интерфейс, объявленный как private interface Interface { ... } - это НЕКОРРЕКТНОЕ объявление и вызовет ошибку компиляции. Причины:

Интерфейсы не могут быть private

Интерфейсы могут быть только: public или package-private (default)

Private интерфейсы не имеют смысла, так как их невозможно реализовать извне

Дополнительные пояснения:

Интерфейсы могут объявляться только на уровне пакета (top-level) или как члены класса (nested)

Вложенные интерфейсы (nested interfaces) могут иметь больше модификаторов доступа (private, protected) в зависимости от контекста

Top-level интерфейсы (объявленные на уровне пакета) могут быть только public или package-private
